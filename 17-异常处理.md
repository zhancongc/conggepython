> 没有完美的人，同样地，也没有完美的代码。在代码运行过程中，难免发生异常。这些异常可能是运行环境的不匹配，也可能是错误的数据输入，或者是代码本身的问题。为了保证代码的正常运行，我们需要探讨怎么处理异常。

# 什么是异常处理

响应大家多举例子的要求，在讲异常处理之前先举个例子。

对于下面的匿名函数，它的作用是求倒数，只要用1除以这个数即可。但是在数学中我们学过，0不可以做分母，因此，这个匿名函数在遇到0时，会出现`ZeroDivisionError`的异常。

```python
# 匿名函数：求一个数的倒数
>>> fun = lambda x : 1/x 
>>> fun(0)
ZeroDivisionError: division by zero
```

如果我们使用if语句来处理这种异常也是可以的。

```python
import sys
def fun(x):
    if x == 0:
        print("ZeroDivisionError")
        # 打印出错误信息
        print(sys.exc_info()[0])
    else:
        return 1/x
>>> fun(0)
```

这样，就可以在异常到来之前防患于未然。不过，这种方法必须提前考虑到异常发生的条件，否则我们没有办法去处理这个异常。在实际编程过程中，很少有一眼就能看出异常发生条件的地方。除此之外，异常的种类繁多，还需要我们根据不同类型的异常给出不同的处理方式。因此，这种用if语句来预测和处理异常的方法并不推荐使用。python内置了try-except结构，专门用于处理异常。上面的异常处理，可以用try-except结构改写成：

```python
def fun(x):
    try:
        return 1/x
    except ZeroDivisionError as e:
        print(e)
# 当分母是0的时候，抛出异常信息
>>> fun(0)
division by zero
# 当分母不是0的时候，正常输出结果
>>> fun(10)
0.1
```

这里的 try-except结构的运行流程是：尝试运行代码`return 1/x`，如果没有出错，则正常运行至程序结束。如果出现了`ZeroDivisionError`，则打印出错误信息。这里，我们能够轻易预见到是`ZeroDivisionError`，如果不能预见是`ZeroDivisionError`，可以用`Exception`代替，这样就可以捕获所有常见的异常。

不过，不推荐大家使用`Exception`，因为`Exception`不够精确，不能够发挥try-except结构根据不同异常给出不同处理方式的优势。

## 异常处理的流程

根据上面的例子，我们不难归纳出异常处理的一般形式：

```python
try:
    代码块A
except 异常类型1:
    代码块1
except 异常类型2:
    代码块2
```

其中，代码块A中可能发生异常，当发生异常，并且是异常类型1的时候，执行代码块1中的内容；当发生异常，并且是异常类型2的时候，执行代码块2的内容。

代码块A的代码行数越短，错误就越容易定位。因此，代码块A处的代码越少越好。聪哥建议只要满足以下条件中的一个，就应该放入代码块A：

* 业务部分关键的代码，不能容忍其发生错误
* 不熟悉的、知识尚不能彻底掌握的代码
* 历史上曾多次出错的代码

除了except部分可以对异常作出处理，还有else部分和finally部分也可以对异常进行处理。实际上，异常处理最完整的形式是这样的：

```python
try:
    代码块A
except 异常类型1:
    代码块1
except 异常类型2:
    代码块2
    ...此处省略很多异常类型和处理
except 异常类型n:
    代码块n
else:
    代码块B
finally:
    代码块C
```

如果上面所有的异常都不满足，就会执行else部分的代码块B。因此，发生意料之外的异常时，代码块B可以给出应急的处理方案。

无论是否发生异常，都会执行finally中的代码块C。代码块C经常被用作代码块A执行后的环境清理。比如，在代码块A中打开一个文件，不管有没有发生异常，在处理结束之后都需要关闭文件。

```python
try:
    # 假设当前工作目录下有文本文件class.txt
    f = open('class.txt', 'r', encoding='utf8')
    print([line for line in f])
except IOError as e:
    print("IOError: ", e.value)
else:
    print("open file error")
finally:
    # 使用完文件后必须要关闭文件
    f.close()
```

## 异常还有哪些类型

想要合理地处理异常，就必须要认识异常。异常本质上是类，python有很多内置的标准异常，比如：`KeyError`, `NameError`, `SyntaxError`,` ValueError`等，它们都继承自`BaseException`这个最基本的异常类。

```python
# 分母是0
>>> 1/0
ZeroDivisionError: division by zero
# 未事先定义number就使用
>>> number + 1
NameError: name 'number' is not defined
# if语句中的代码没有缩进
>>> if number > 0:
>>> print(number)
IndentationError: expected an indented block
```

下面是一些常见的异常和出现的原因

| 异常                | 原因          |
| ----------------- | ----------- |
| BaseException     | 所有异常的基类     |
| Exception         | 常规异常的基类     |
| ImportError       | 导入模块或对象失败   |
| IOError           | 输入输出错误      |
| KeyboardInterrupt | 用户中断执行      |
| ZeroDivisionError | 分母是0        |
| ValueError        | 传入无效的参数     |
| TypeError         | 待操作的变量的类型错误 |
| NameError         | 没有这个变量      |
| IndexError        | 列表没有这个索引    |
| KeyError          | 字典中没有这个键    |
| AttributeError    | 对象没有这个属性    |
| SyntaxError       | 语法错误        |
| IndentationError  | 缩进错误        |

## 主动抛出异常

有的时候，代码符合python的要求，但是不符合业务的要求。比如：一门课程的成绩只能是正数，如果分数是一个负数，虽然属于异常却合乎python语法，因而python并不会主动抛出异常。此时，就需要我们主动抛出异常。主动抛出异常需要使用raise语句。

```python
# 一旦分数小于0，则抛出异常，中断代码执行
>>> score = -40
>>> if score < 0:
···    	raise ValueError("score cannot be negative")
ValueError: score cannot be negative
```

主动抛出异常，就会中断代码的执行，避免一错再错，保障了程序安全。如果不考虑异常而强行运行代码，就会发生不可预知的后果，甚至造成难以估量的损失。

## 自定义异常

虽然前面我们介绍了很多种异常，但是在实际编程过程中，仍然不能满足各种业务需求。这个时候，除了能够主动抛出python内置的异常并不妥当，而自定义异常就能很好地满足我们的需求。前面我们曾提到，异常的本质是类，所有的异常的基类是`BaseException`，但是我们遇到的基本上是常规异常`Exception`。因此，自定义异常实际上就是自定义一个类，它继承自`Exception`。

下面给出了一个如何使用自定义异常的例子：

```python
class NegativeError(Exception):
    def __init__(self, value):
        self.value = value
    def __str__(self):
        # str()方法将值转化为适于人阅读的形式，而repr()将值转化为供解释器读取的形式。
        return repr(self.value)
>>> score = -40
>>> try:
>>>     if score < 0:
>>>         raise NegativeError("score cannot be negative")
>>> except NegativeError as e:
>>>     print('NegativeError: ', e)
NegativeError: score cannot be negative
```

只要score小于0，都会触发`NegativeError`，我们使用raise语句抛出异常，然后用try-except结构接收和处理异常。

## 总结

1. 代码运行难免出错，抛出异常是为了代码健壮性的表现
2. 在try-except结构中，可能出错的代码放在try部分，满足异常条件则执行对应对应的except部分的代码，没有符合的异常则执行else部分的代码，无论是否发生异常都执行finally部分的代码
3. 异常的本质是类，需要记住常见的python内置异常
4. 根据业务需要，使用raise语句主动抛出异常
5. python内置的异常不满足的时候，需要自定义异常
